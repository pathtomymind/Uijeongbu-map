프롬프트 1 (최적화 순회 경로를 표로 받기 위한 프롬트프)

제미나이, 너는 건축과 도시정책, 주택 관련 전문 행정가로 도시의 건축, 토목, 도시계획, 민원 행정 처리의 전문가야. 의정부 시청 건축과 도시미관팀에서 불법 옥외광고물 정비를 하기 위해 의정부시를 순회할 거야. 국민신문고 등에서 취합한 민원 요청 사항을 정리한 표를 이미지로 촬영해서 함께 올릴께. 이 이미지들을 분석해서 의정부 시청에서 출발하여 민원 요청 지역을 순회하고 복귀하는, 최적화된 순회 경로를 만들어줘. 표 형식으로 만들어줘. 순회 경로를 만드는 기준은 단순히 방문할 지점들의 좌표간 직선 거리 기준이 아니고, 지그재그로 이동하는 동선 낭비를 하지 않기 위해 동네를 그룹화하고, 의정부의 주요 도로 연결성을 고려하고, 출발지와 도착지가 같기 때문에 한 방향으로 크게 원을 그리며 도는 루프형 동선으로 설계해줘. 이 표의 제목은 <오늘의 최적화 순회 경로 표>로 해줘. 이 표의 컬럼은 다음과 같이 구성해 줘.
* 체크 박스 (표 제목은 필요 없고 작업할 때 완료 여부를 표시하는 용도)
* 방문 순서 (일련번호)
* 원래 연번 (이미지로 올린 민원 요청 사항에 있던 원래 일련번호)
* 주소 (주소 표시 후 행을 바꿔서 지역 설명을 괄호로 넣어줘)
* 민원 내용
* 현장 사진 요약 (업로드한 이미지의 비고란 내용과 현장 사진의 내용을 분석해서 찾기 쉬운 특징을 짧게 표현)
그리고 표 아래에는 구성한 동선에 대한 간략한 설명을 짧게 붙여줘.
표의 헤더행과 데이터행이 어긋나지 않도록 줄바꿈 문자를 없애줘.

프롬프트 2 (카카오톡 등에 작업자들과 공유하기 위한 이미지 생성 요청)

이 표를 카카오톡에 공유하고 싶어. 표 전체를 그대로 이미지로 만들어줘. 모든 컬럼이 다 나오게 해주고 문서의 여백은 1센티 정도로 작게 해줘. 표를 이미지로 만들 때 임의로 글자를 조정하지 말고 표에 있는 글자를 그대로 유지해줘.

프롬프트 3 (실시간 지도 업데이트용 JSON 생성)

위에서 작성한 '오늘의 최적화 순회 경로 표'를 바탕으로, 내 깃허브 지도(index.html)에서 호출하는 data.json 파일의 내용을 생성해 줘. 작성 시 아래의 기술적 규칙을 반드시 엄격하게 준수해야 해:
* JSON 구조: 반드시 { "points": [ { "lat": 위도, "lng": 경도, "name": "순번. 지점명" }, ... ] } 형식을 유지할 것.
* 좌표값: 의정부시 관내 지리 정보에 기반하여 각 주소지의 위도(lat)와 경도(lng)를 소수점 4자리까지 정확히 추론하여 입력할 것.
* 기점 설정: 첫 번째 포인트와 마지막 포인트는 반드시 **의정부시청(lat: 37.7381, lng: 127.0333)**으로 설정할 것.
* 필드 매핑: name 필드에는 '방문 순서. 지점 이름(동네명)' 형식으로 간결하게 작성할 것 (예: "2. 가능동 RC사거리").
* 출력 형태: 스마트폰에서 즉시 복사하여 깃허브 파일에 붙여넣을 수 있도록, 일체의 설명이나 인사말 없이 순수 JSON 코드 블록만 출력할 것.
그리고 마지막에 다음의 링크를 만들어줘. 이 링크를 복사해서 카카오톡으로 보내기 위함이야. 
실시간 지도: https://pathtomymind.github.io/Uijeongbu-map/
