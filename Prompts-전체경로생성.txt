🗺️ [저장용] 전체 경로(Full Route) 시각화 프롬프트
[과업: 정비 경로 지도 시각화] 위에서 추출한 데이터를 바탕으로 Leaflet 및 Leaflet Routing Machine 라이브러리를 사용한 '전체 경로 시각화 HTML'을 생성하라.

1. 시각화 및 디자인 가이드라인

경로 표시: 기점(의정부시청)에서 마지막 정비 지점까지는 파란색 실선으로, 마지막 지점에서 종점(의정부시청)까지는 빨간색 점선으로 표시하여 복귀 동선을 구분한다. 모든 경로는 직선(Straight lines)이 아닌 실제 도로 주행 경로(Road-following path)를 반영해야 하며, 이를 위해 Leaflet Routing Machine을 필수적으로 사용하라

UI 정리: 우측 상단의 경로 요약 박스(Control Panel)나 범례 등 지도를 가리는 모든 패널을 제거하여 전체 화면(100vh)을 확보한다.

상시 정보 노출: 각 지점 마커 옆에는 최적화 경로의 순번과 지점제목이 포함된 말풍선(Tooltip)이 클릭 없이도 항상 보이도록(permanent: true) 설정한다.  

2. 출력 및 파일 관리 규칙

URL 선출력: HTML 코드 블록 생성 직전에, 깃허브 배포 주소인 https://pathtomymind.github.io/Uijeongbu-map/(파일명) 형식을 가장 먼저 표시한다.

파일명: YYYY-MM-DD_Full_Route_Vn.html (n은 생성 일련번호)로 지정한다.

미리보기 구현: 카카오톡 공유 시 제목과 설명이 하얀 박스로 예쁘게 뜨도록 <head> 내에 Open Graph 태그를 포함한다.

[추가 지침: 도로망 최적화 및 시각적 개선]

도로 기반 경로: 단순 직선(L.polyline)이 아닌 Leaflet Routing Machine과 OSRM 엔진을 사용하여 실제 도로망을 따라 선을 그려라.

로컬 도로 우선: 고속화도로(신평화로 등)로의 비합리적인 우회를 방지하기 위해, POI 좌표 생성 시 실제 차량 진입로에 가깝게 보정하고 가급적 최단 거리의 로컬 도로를 우선하라.

말풍선 겹침 방지: 기점과 종점이 같은 경우, 1번 출발 지점의 툴팁은 direction: 'top', 마지막 복귀 지점의 툴팁은 direction: 'bottom'으로 설정하여 글자가 겹치지 않게 하라.
